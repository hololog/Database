<트랜잭션 개념>
-하나의 논리적인 작업 단위
-여러개의 DML이 하나의 트랜잭션
-하나의 DDL, DCL이 하나의 트랜잭션
---------------------------------------------
트랜잭션 시작 ->		insert ---;
			update ---;
			update ---;
트랜잭션 종료 ->		commit;
----------------------------------------------
트랜잭션 시작 ->		delete ---;
			delete ---;
트랜잭션 종료 ->		rollback;
----------------------------------------------
트랜잭션 시작/종료 ->	create table ---;(DDL)
			(autocommit 내포)
----------------------------------------------
트랜잭션 시작/종료 ->	grant ---;(DCL)
			(autocommit 내포)
----------------------------------------------
트랜잭션 시작 ->		update ---;
			insert ---;
			create table ---;(DDL)
트랜잭션 종료 ->		(autocommit)
----------------------------------------------
트랜잭션 시작 ->		update ---;
			delete ---;
			delete ---;
			(커밋/롤백을 수행하지 않고, 프로그램을 정상 종료)
트랜잭션 종료 ->		(autocommit)
----------------------------------------------
트랜잭션 시작 ->		update ---;
			delete ---;
			delete ---;
			(커밋/롤백을 수행하지 않고, 프로그램을 비정상 종료)
트랜잭션 종료 ->		(autorollback)
----------------------------------------------
